## Sesion — Migracion Astro 6 a Vite + React + React Three Fiber

### Commits realizados

### Cambios principales

**Problema raiz**: Astro 6 MPA causa bugs de renderizacion WebGL/SVG al navegar entre paginas. El bfcache y Speculation Rules API de Chrome impiden que model-viewer y custom elements se inicialicen correctamente en paginas pre-renderizadas.

**Solucion**: Migracion completa a SPA con Vite + React Router + React Three Fiber.

#### Fase 1: Estructura Vite + React Router
- `vite.config.ts` reemplaza `astro.config.mjs`
- `src/main.tsx` monta React app con BrowserRouter (basename="/galeria-3d-clase")
- `src/App.tsx` define rutas: /, /estudiantes, /perfil
- `src/layouts/Layout.tsx` reemplaza Layout.astro con NavLink + Outlet

#### Fase 2: Migracion de componentes
- Todos los 13 componentes React migrados sin cambios funcionales
- `supabase.ts` reutilizado intacto
- Pages Astro convertidas a React page components

#### Fase 3: React Three Fiber reemplaza model-viewer
- `Model3D.tsx` — Carga GLB con useGLTF, centrado automatico (scale -> recalc bbox -> center), fix color space NoColorSpace para non-color maps
- `ModelScene.tsx` — Escena reutilizable: Environment studio (0.4), ambient (0.15), dual directional lights, suelo mate #222, ContactShadows, fog
- `ModelCard.tsx` — Canvas con frameloop="demand"/"always" segun hover
- `ModelModal.tsx` — Canvas con controles completos (orbit, zoom, pan)
- `UploadForm.tsx` — Preview R3F en vez de model-viewer

### Decisiones tecnicas clave
- **frameloop="demand"** en tarjetas: 0 fps GPU cuando no hay hover, vital para 13+ modelos simultaneos
- **Scale first, then center**: Box3 centrado requiere scale -> updateMatrixWorld -> recalc box -> sub(center)
- **NoColorSpace** para normal/roughness/metalness/AO maps (Blender exporta OpenGL)
- **sessionStorage redirect** para SPA routing en GitHub Pages (mas simple que query string approach)
