## Sesión — Diseño RBAC + Entorno local (Fase 0)

### Contexto

Continuación de la migración completa Supabase → DigitalOcean (13/04). Objetivo de esta sesión: diseñar el sistema multi-rol (admin/teacher/student) con relación profesor↔estudiante, y preparar entorno de desarrollo local para prototipar la migración DB antes de tocar producción.

### Decisiones de producto

- **Registro restringido a `@unbosque.edu.co`** — dominio institucional obligatorio a nivel DB (CHECK constraint) y app.
- **Roles independientes** — admin y teacher son roles distintos. Un usuario puede tener ambos (Carlos = admin+teacher).
- **Multi-rol por usuario** — tabla pivote `user_roles` (M:N) reemplaza columna `profiles.role`.
- **Relación N:M profesor↔estudiante** — tabla `teacher_students` en vez de `profiles.teacher_id`. Soporta cohort/talleres futuros.
- **Visibilidad**: todos ven la galería; solo el profesor asignado (o el admin) puede gestionar modelos de un estudiante.
- **Flujo de creación**:
  - Estudiantes: autoregistro con correo `@unbosque.edu.co`.
  - Docentes: creados por admin.

### Informe técnico — Diego Ramírez (Data Lead)

Revisión DBA del schema propuesto. Ocho mejoras críticas:

1. `roles.id SMALLINT` con IDs explícitos (no SERIAL) para que dev/prod no diverjan.

**5. Validación del fix UTF-8 — dos vueltas**

- Primer intento: re-envío con curl desde Git Bash mostró mojibake persistente (`Galer�a`, `contrase�a`) **incluso en el subject**. Eso descartaba el meta tag como causa — el problema venía desde antes.
