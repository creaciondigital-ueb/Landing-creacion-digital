### Cambios principales

#### 1. Auditoría completa del proyecto (mesa de desarrollo)
Sebastián Torres Mejía ejecutó auditoría del stack completo. Diagnóstico de 5 problemas raíz.

#### 2. Fix: loading infinito en todas las páginas (try/catch)
`loadData()` / `loadModels()` / `load()` no tenían `try/catch`. Si Supabase fallaba, `setLoading(false)` nunca corría. Fix: `try/catch/finally` en `EstudiantesPage.tsx`, `Gallery.tsx`, `ProfilePage.tsx`.

#### 3. Fix: race condition auth Supabase v2
Supabase v2 encola todas las queries mientras refresca el token. Queries lanzadas antes de `await getSession()` nunca se envían. Fix: `init()` con `await getSession()` primero, luego queries. Aplicado en `Gallery.tsx` y `EstudiantesPage.tsx`.

#### 4. Fix: modal de login descentrado
`#top-bar` tiene `backdrop-filter: blur(12px)` que crea containing block para `position: fixed`. Modal quedaba posicionado relativo al top-bar (44px). Fix: `createPortal(content, document.body)` en `AuthModal.tsx`.

#### 5. Fix: ClientRouter (ViewTransitions) eliminado
`ClientRouter` de `astro:transitions` es incompatible con múltiples React islands `client:load`. Los componentes no se rehidratan en el swap de página → pantallas en blanco. Fix: removido de `Layout.astro`.

#### 6. Fix: mutación directa de prop en StudentCard
`handleSaveLinks()` mutaba `student.artstation_url` directamente. React no detecta mutaciones → vista no se actualizaba. Fix: vista usa estado local `artstation`/`instagram`.

#### 7. Fix: getUserProfile() retornaba undefined
`.single()` retorna `undefined` sin fila. `return data ?? null` previene undefined.

#### 8. Fix: comentarios 400 Bad Request
`.select('*, profiles(full_name, role)')` falla porque FK `comments_user_id_fkey` apunta a `auth.users` no a `public.profiles`. Fix: dos queries separadas — `select('*')` en comments + fetch de profiles por `user_id[]`.

#### 9. Vite cache corruption
504 "Outdated Optimize Dep" — `rm -rf node_modules/.vite` + `npm run dev -- --force`.
